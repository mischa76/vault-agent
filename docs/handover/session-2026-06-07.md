## Lessons Learned aus den letzten Setup-Schritten
1. **Docstring muss erste Anweisung im File sein** – sonst wird er zur Expression und
   Ruff (\`E402\`) meckert. Reihenfolge: Docstring → stdlib → 3rd-party → lokal.
2. **Strict mypy verlangt Type-Args überall** – \`dict[str, Any]\` statt nacktem \`dict\`,
   \`StateGraph[VaultAgentState]\` statt nacktem \`StateGraph\`. Konsequent durchhalten.
3. **Beim Editieren von Pydantic-Modellen** Forward-References checken – wenn \`VaultAgentState\`
   auf \`BusinessKeyCandidate\` zeigt, muss die Klasse definiert sein, bevor \`VaultAgentState\`
   geparsed wird. Reihenfolge zählt.
4. **Vor jedem Commit \`git status\` + \`git diff --staged\`** – verhindert kaputte Pushes.
5. **\`code\` in WSL ist Windows-VS-Code via WSL-Extension**, nicht Snap-Install.
