# Data Contracts – Vault-Agent's Approach

> Based on Andrew Jones, "Driving Data Quality with Data Contracts" (Manning).

## Where contracts live
At the source-to-staging boundary. Generated by the Data Contract Agent from the
identified entities and their inferred semantics.

## What's in a contract
- Schema (fields, types, nullability)
- Semantic definitions (what each field means in business terms)
- SLAs (freshness, completeness, accuracy)
- Ownership (who owns the source, who consumes it)
- Versioning policy

## How they're enforced
Runtime: a pre-load validation step in the dbt pipeline rejects rows that don't conform.
Tests: dbt test layer derived from the contract.
